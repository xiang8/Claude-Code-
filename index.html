▶️ 启动 Claude Code

📂 如何使用 Claude Code？

Claude Code 需要在某个文件夹里面启动，它才能帮你处理这个文件夹里的文件

启动步骤（一步一步）
  1. 找一个你想用来写代码的文件夹（可以在桌面新建一个）
  2. 进入这个文件夹（双击打开它）
  3. 在文件夹窗口顶部的地址栏输入：cmd
  4. 按 回车键，会出现黑色的命令行窗口
  5. 在这个窗口中输入：claude，然后按回车
  6. 看到 Claude Code 界面就启动成功了！
💡 小技巧

以后想在这个文件夹用 Claude Code，只需要：进入文件夹 → 在地址栏输入 cmd → 输入 claude

⚠️ 启动时提示需要 git-bash？

如果提示「需要 git-bash」，需要设置环境变量：

步骤0：先找到 Git 安装位置

在命令行中运行：

where git

会显示类似：C:\Program Files\Git\cmd\git.exe

把 cmd\git.exe 改成 bin\bash.exe 就是你要的路径

方法1：PowerShell 命令（推荐）

[System.Environment]::SetEnvironmentVariable('CLAUDE_CODE_GIT_BASH_PATH', 'C:\Program Files\Git\bin\bash.exe', 'User')

然后重启命令行窗口

方法2：图形界面设置

  1. 按 Win + R，输入 sysdm.cpl，回车
  2. 点击「高级」→「环境变量」
  3. 在「用户变量」中点击「新建」：
    • 变量名：CLAUDE_CODE_GIT_BASH_PATH
    • 变量值：C:\Program Files\Git\bin\bash.exe
  4. 点击「确定」保存，重启命令行窗口

💡 如果 Git 安装在其他位置，请修改路径

配置 MCP 服务器

🔌 什么是 MCP？

MCP（Model Context Protocol）是 Claude Code 的扩展功能，可以添加各种实用工具（如搜索、看图、读网页等）

💡 配置建议：全部开启

MCP 服务器只在需要时才启动，用完会自动停止，不会占用系统资源

在配置过程中，遇到 MCP 服务器选项时：全部选择 Yes/启用

  • ✅ 搜索 MCP - 网络搜索功能
  • ✅ 视觉 MCP - 识别图片功能
  • ✅ 网页读取 MCP - 抓取网页内容
  • ✅ 其他可用的 MCP 工具
💡 如何确认 MCP 已开启？

配置完成后，在 Claude Code 中输入 /mcp 查看已启用的 MCP 服务器

🧠 最重要的一步：建立记忆档案
⚠️ 为什么这一步很重要？

装了 Claude Code 但没做这一步，你还是在用「失忆的临时工」。

做完这一步，它才能记住你是谁、你在做什么、你喜欢什么风格——永远不用再自我介绍第二遍。

创建你的 CLAUDE.md 记忆档案

在 Claude Code 界面里，复制粘贴下面这段话（把括号里的内容替换成你自己的）：

我叫[你的名字]，我在做[你的工作方向，比如：自媒体运营/产品经理/独立顾问]。 我的主要日常任务有： - [任务1，比如：写微信公众号文章] - [任务2，比如：管理客户项目] - [任务3] 我的工作风格偏好：[比如：简洁直接，不要废话 / 需要详细解释每个步骤] 我常用的工具：[比如：Notion、飞书、微信] 请帮我创建一个 CLAUDE.md 文件，把以上信息记录下来，作为我们长期合作的基础档案。以后每次我打开 Claude Code，你都要先读这个文件，了解我的背景再开始工作。

它会在你的工作文件夹里生成一个 CLAUDE.md 文件。这就是它的「记忆档案」。

💡 这个文件的作用

每次启动 Claude Code 后，它会自动读取 CLAUDE.md，了解你是谁、在做什么、偏好什么风格，然后再开始工作。

你再也不用每次重新介绍自己了！

📂 重要：确保每次都读取

两个关键点：

  1. CLAUDE.md 必须放在工作文件夹的根目录
  2. 启动 Claude Code 时必须在这个文件夹中

❌ 错误示例

在 D:\ 工作，但 CLAUDE.md 在 D:\工作\项目\ 里 → 读取不到

✅ 正确示例

进入 D:\工作\项目\ 文件夹，然后启动 Claude Code → 自动读取

🔍 验证是否生效

创建完 CLAUDE.md 后，直接问它：

你读过 CLAUDE.md 了吗？我是谁？

如果它能正确回答你的名字和工作，说明生效了 ✅

⚠️ 如果不生效怎么办？

有时 Claude Code 可能没有自动读取，手动告诉它：

请先读取 CLAUDE.md 文件，了解我的背景再开始工作

❓ 常见问题
⚠️ 安装失败？90% 的坑在这里

安装过程中遇到报错不要慌，以下是最常见的三大坑和对应解法：

🪤 坑 #1：进度条不动 / ETIMEDOUT 报错

这是最常见的问题。原因是终端（黑框框）没有走代理，即使你的浏览器能开 Claude，终端依然走的是直连。

Windows 用户：

  • 确保代理软件开启全局模式
  • 使用美国节点（或其他非被封锁的节点）
  • 检查代理端口是否正确（常见端口：7890、10809 等）

✓ 解法： 确认代理设置正确后，重新运行安装命令

🪤 坑 #2：提示找不到命令 / command not found

安装完毕后输入 claude 提示找不到命令，通常是 PATH 没有生效。

✓ 解法： 关掉终端窗口，重新打开一个新的终端，再输入 claude 试试

🪤 坑 #3：浏览器弹出但授权失败

点了 Authorize 之后没有反应，或者跳回报错页面。通常是：

  • 登录的 Claude 账号不是 Pro 会员
  • 网络在授权跳转时断了
  • 代理没开全局模式

✓ 解法： 确认 claude.ai 账号是 Pro，代理开全局模式，关掉浏览器重试

💡 万能排错思路

"我在安装 Claude Code，遇到这个报错，帮我分析原因：[粘贴报错内容]"

Q: Windows 脚本报错怎么办？

A: 在 PowerShell 中运行以下命令允许脚本执行：

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
Q: 启动时提示需要 git-bash？

A: 设置环境变量 CLAUDE_CODE_GIT_BASH_PATH：

[System.Environment]::SetEnvironmentVariable('CLAUDE_CODE_GIT_BASH_PATH', 'C:\Program Files\Git\bin\bash.exe', 'User')

然后重启命令行窗口

Q: 配置修改不生效？

A: 尝试以下步骤：

  • 关闭所有 Claude Code 窗口
